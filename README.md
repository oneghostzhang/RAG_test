# 職能基準知識圖譜 Graph RAG 系統

基於台灣 ICAP 職能基準的知識圖譜系統，整合 Graph RAG 技術，支援跨職業能力比較與職涯路徑規劃查詢。

## 🎯 系統特色

### 核心功能

- **📄 PDF 自動解析**
  - 自動解析 ICAP 職能基準 PDF 文件
  - 提取職能代碼、工作任務、知識、技能、態度等結構化資料
  - 支援批次處理數千份 PDF

- **🕸️ 知識圖譜建構**
  - 11 種節點類型（職能基準、職類別、知識、技能等）
  - 13 種邊類型（屬於、包含、需要等關係）
  - 自動推斷職涯晉升路徑和知識/技能關聯

- **🔍 五種查詢類型**
  1. **跨職業比較**: 比較兩個職業的共同和差異能力
  2. **職涯路徑**: 找出晉升路徑和需要補充的能力
  3. **能力反查**: 根據知識/技能找適合的職業
  4. **聚合統計**: 統計最常被需要的知識/技能
  5. **語義搜尋**: 自然語言查詢 + 圖譜擴展

- **🔗 混合檢索策略**
  - 向量檢索：對描述文字做 embedding 相似度搜尋
  - 圖譜遍歷：根據檢索結果沿著邊擴展相關節點
  - 結果融合：結合向量分數 + 圖譜路徑權重排序

## 🚀 快速開始

### 1. 安裝依賴

```bash
cd C:\Users\User\Graph_RAG_test
pip install -r requirements.txt
```

### 2. 執行測試

```bash
python quick_start.py --mode test --limit 5
```

這會：
- ✅ 測試 PDF 解析器
- ✅ 建構測試知識圖譜（5 個職能基準）
- ✅ 測試 Graph RAG 查詢

### 3. 建構完整圖譜

```bash
# 建構 100 個職能基準的圖譜
python quick_start.py --mode build --limit 100

# 建構全部（約 2,882 個，需要較長時間）
python quick_start.py --mode build
```

### 4. 互動查詢

```bash
python quick_start.py --mode interactive
```

## 📊 知識圖譜結構

### 節點類型

| 節點類型 | 屬性 | 說明 |
|----------|------|------|
| 職能基準 | 代碼, 名稱, 工作描述, 基準級別 | 如：TFB7912-002v3 西點麵包烘焙助理 |
| 職類別 | 代碼, 名稱 | 如：TFB 休閒與觀光旅遊/餐飲管理 |
| 職業別 | 代碼, 名稱 | 如：7912 麵包點心及糖果製造人員 |
| 行業別 | 代碼, 名稱 | 如：C0891 烘焙炊蒸食品製造業 |
| 主要職責 | 代碼, 名稱 | 如：T1 烘焙前置準備 |
| 工作任務 | 代碼, 名稱, 職能級別 | 如：T1.1 整理環境與設備 (級別2) |
| 工作產出 | 代碼, 名稱 | 如：O1.1.1 環境維護紀錄表 |
| 行為指標 | 代碼, 描述 | 如：P1.1.1 依食品安全衛生規範... |
| 知識 | 代碼, 名稱, 描述 | 如：K01 食品安全衛生相關規範 |
| 技能 | 代碼, 名稱, 描述 | 如：S01 器具選用及操作能力 |
| 態度 | 代碼, 名稱, 描述 | 如：A01 主動積極 |

### 邊類型

| 起點 | 關係 | 終點 |
|------|------|------|
| 職能基準 | 屬於職類 | 職類別 |
| 職能基準 | 屬於職業 | 職業別 |
| 職能基準 | 適用行業 | 行業別 |
| 職能基準 | 包含職責 | 主要職責 |
| 主要職責 | 包含任務 | 工作任務 |
| 工作任務 | 產出 | 工作產出 |
| 工作任務 | 要求行為 | 行為指標 |
| 工作任務 | 需要知識 | 知識 |
| 工作任務 | 需要技能 | 技能 |
| 職能基準 | 要求態度 | 態度 |
| 知識 | 相關於 | 知識 |
| 技能 | 前置於 | 技能 |

## 🔧 模組說明

### config.py
系統配置管理，包含：
- 路徑配置
- LLM/Embedding 模型設定
- 節點和邊類型定義
- 正則表達式編碼格式

### pdf_parser.py
PDF 解析模組：
- 使用 PyMuPDF 提取文字和表格
- 解析職能基準代碼（TFB7912-002v3）
- 解析工作任務（T1.1）、產出（O1.1.1）、行為指標（P1.1.1）
- 解析知識（K01）、技能（S01）、態度（A01）
- 輸出結構化 JSON

### graph_builder.py
知識圖譜建構模組：
- 使用 NetworkX 建立有向多重圖
- 建立節點和邊索引
- 推斷職涯晉升路徑
- 推斷知識/技能關聯
- 圖譜序列化儲存

### graph_rag.py
Graph RAG 查詢引擎：
- 使用 FAISS 建立向量索引
- 實現五種查詢類型
- 混合檢索策略
- 可選 LLM 答案生成

## 🛠️ 技術棧

| 組件 | 技術 | 用途 |
|------|------|------|
| PDF 解析 | PyMuPDF | 文字和表格提取 |
| 知識圖譜 | NetworkX | 圖資料結構 |
| 向量檢索 | FAISS | 高效相似度搜尋 |
| Embedding | sentence-transformers | 文本向量化 |
| LLM | llama-cpp-python | 答案生成（可選） |

## ⚠️ 注意事項

1. **PDF 解析精度**
   - 依賴 PDF 格式的一致性
   - 複雜排版可能影響準確度
   - 建議檢查解析結果

2. **記憶體使用**
   - 完整圖譜可能需要 2-4GB
   - Embedding 模型約 500MB
   - 建議系統記憶體 ≥ 8GB

3. **資料來源**
   - 職能基準資料來自 [ICAP 職能發展應用平台](https://icap.wda.gov.tw/)
   - 僅供學習研究使用
